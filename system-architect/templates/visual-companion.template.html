Option A
Plugin Registry Pattern Recommended
Named handlers registered at startup, looked up at runtime.
Interface
interface Plugin {
  name: string;
  execute(ctx: Context): Promise<Result>;
}

const registry = new Map<string, Plugin>();

function register(plugin: Plugin) {
  registry.set(plugin.name, plugin);
}

function run(name: string, ctx: Context) {
  const plugin = registry.get(name);
  if (!plugin) throw new Error(`Unknown plugin: ${name}`);
  return plugin.execute(ctx);
}
Architecture
flowchart TD
    App -->|register| Registry[(Plugin Registry)]
    App -->|run name| Registry
    Registry -->|lookup + call| PluginA[Plugin A]
    Registry -->|lookup + call| PluginB[Plugin B]
Tradeoffs

Pros

  • Simple — one Map, no loader
  • Testable in isolation
  • No new infrastructure

Cons

  • Plugins must be known at startup
  • No dynamic loading
Option B
Dynamic Loader + Version Resolver
Plugins discovered from disk at runtime with semver resolution.
Interface
interface PluginManifest {
  name: string;
  version: string;
  entrypoint: string;
}

class PluginLoader {
  async load(name: string, range: string): Promise<Plugin> {
    const manifest = await this.resolve(name, range);
    const mod = await import(manifest.entrypoint);
    return mod.default;
  }

    // scan plugin dirs, parse manifests, semver match

Architecture
flowchart TD
    App -->|load name@range| Loader[Plugin Loader]
    Loader -->|scan| Disk[(Plugin Directories)]
    Loader -->|parse| Manifest[Manifests]
    Loader -->|semver match| Resolver[Version Resolver]
    Resolver -->|import| Plugin[Plugin Module]
Tradeoffs

Pros

  • Plugins installable without restart
  • Version pinning per consumer

Cons

  • 3 layers for a problem we don't have yet
  • Disk I/O on every load
  • Semver resolver is non-trivial

Notes

  • Option A is sufficient unless plugins need to be added without a restart — that requirement hasn't been stated.
  • Option B's complexity is justified only when plugins come from untrusted third parties with incompatible versions.
  • Open question: do plugins need to be composable (chained), or are they always invoked independently?
